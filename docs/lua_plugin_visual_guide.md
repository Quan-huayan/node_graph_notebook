```

│                          ▼                                    │

│  │ >(_print, 0)                                         │    │
│  └──────────────────────────────────────────────────────┘    │
└───────────────────────────────────────────────────────────────┘
                          │
                          ▼
┌───────────────────────────────────────────────────────────────┐
│  dart:ffi (FFI 库)                                            │
│  ┌──────────────────────────────────────────────────────┐    │
│  │ 创建 C 函数指针                                        │    │
```

## 3. Lua 栈操作详解

### 调用 print("Hello", 123) 时的栈状态

```
初始状态:
┌─────────────┐
│   (空)      │
└─────────────┘

压入参数后:
┌─────────────┐
│   123       │  ← 索引 2 (lua_tostring(L, 2) = "123")
├─────────────┤
│  "Hello"    │  ← 索引 1 (lua_tostring(L, 1) = "Hello")
├─────────────┤
│   (更多)    │
└─────────────┘

操作完成后:
┌─────────────┐
│   (空)      │  ← 栈被清空
└─────────────┘
```

### 栈操作 API

```dart
// 1. 获取栈大小
int top = lua_gettop(L);  // 返回栈中元素数量

// 2. 获取指定索引的值类型
int type = lua_type(L, index);
// LUA_TNIL, LUA_TBOOLEAN, LUA_TNUMBER, LUA_TSTRING, etc.

// 3. 获取字符串值
Pointer<Char> str = lua_tolstring(L, index, nullptr);
String dartStr = str.cast<Utf8>().toDartString();

// 4. 获取布尔值
int boolVal = lua_toboolean(L, index);
bool dartBool = boolVal != 0;

// 5. 压入值到栈
lua_pushnil(L);              // 压入 nil
lua_pushstring(L, cstr);     // 压入字符串
lua_pushnumber(L, value);    // 压入数字

// 6. 设置栈大小
lua_settop(L, 0);  // 清空栈
```

## 4. 双向通信流程

### Dart → Lua (执行脚本)

```
Dart 代码                    Lua 引擎
    │                            │
    │ executeString("print(1)")  │
    ├───────────────────────────>│
    │                            │
    │                            │ 1. 创建 Lua 状态
    │                            │ 2. 编译脚本
    │                            │ 3. 执行脚本
    │                            │
    │                            │ print(1)
    │                            │   │
    │                            │   ▼
    │                            │ 调用 _print C 函数
    │                            │   │
    │                            │   ▼
    │  <输出缓冲区更新>           │ 保存到输出
    │                            │
    │<───────────────────────────│ 返回执行结果
    │                            │
LuaExecutionResult              │
  output: ["1"]                 │
  success: true                 │
```

### Lua → Dart (调用 API)

```
Lua 脚本                    Dart 代码
    │                            │
    │ createNode("Title", ...)   │
    ├───────────────────────────>│
    │                            │
    │                            │ 1. 查找函数指针
    │                            │ 2. 从栈提取参数
    │                            │    "Title", ...
    │                            │ 3. 调用 Dart 函数
    │                            │
    │                            │ createNode(args)
    │                            │   │
    │                            │   ▼
    │                            │ nodeRepository.save()
    │                            │   │
    │                            │   ▼
    │                            │ 异步完成
    │                            │
    │                            │ invokeCallback()
    │  <────────────────────────── │
    │                            │
    │ onCreateComplete(true, ...) │
    │   │                         │
    │   ▼                         │
    │ print("创建成功")            │
```

## 5. 类型转换流程

### Lua 表 → Dart Map/List

```
Lua 脚本中的表:
local data = {
  name = "Alice",
  age = 30,
  tags = {"red", "blue"}
}

转换过程:
┌─────────────────────────────────────────────────────┐
│ 1. 检测类型: lua_type(L, index) == LUA_TTABLE      │
└─────────────────────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────────────┐
│ 2. 尝试解析为数组:                                   │
│    for i = 1, ...                                    │
│      lua_rawgeti(L, index, i)                       │
│      if lua_type(L, -1) == LUA_TNIL                │
│        break  // 不是数组                            │
└─────────────────────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────────────┐
│ 3. 解析为 Map:                                       │
│    lua_pushnil(L)      // 第一个 key                 │
│    while lua_next(L, index) != 0:                  │
│      key = lua_tostring(L, -2)  // 获取 key         │
│      value = _toDart(L, -1)    // 获取 value       │
│      result[key] = value                            │
│      lua_pop(L, 1)         // 弹出 value，保留 key  │
└─────────────────────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────────────┐
│ 4. 递归处理嵌套表:                                   │
│    tags = {"red", "blue"}  →  ["red", "blue"]       │
└─────────────────────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────────────┐
│ Dart 对象:                                           │
│ {                                                   │
│   "name": "Alice",                                   │
│   "age": 30,                                         │
│   "tags": ["red", "blue"]                           │
│ }                                                   │
└─────────────────────────────────────────────────────┘
```

## 6. 内存管理

### 指针生命周期

```
创建函数指针:
┌─────────────────────────────────────────────────────┐
│ 1. Dart 函数定义                                     │
│    int _print(lua_State* L) { ... }                 │
└─────────────────────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────────────┐
│ 2. 转换为 C 函数指针                                 │
│    Pointer<Function> ptr =                          │
│      Pointer.fromFunction(_print, 0)                │
└─────────────────────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────────────┐
│ 3. 保存引用防止 GC                                   │
│    _functionPointers.add(ptr)  ← 重要!             │
└─────────────────────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────────────┐
│ 4. 注册到 Lua                                        │
│    lua_register(L, "print", ptr)                    │
└─────────────────────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────────────┐
│ 5. 使用完毕后释放                                    │
│    dispose():                                       │
│      _functionPointers.clear()                      │
│      // 指针可以被 GC                               │
└─────────────────────────────────────────────────────┘
```

### C 字符串内存管理

```
错误示例 (内存泄漏):
lua_pushstring(L, "Hello".toNativeUtf8());
// ↑ C 字符串永远不会被释放!

正确示例:
final cstr = "Hello".toNativeUtf8();
try {
  lua_pushstring(L, cstr.cast<Char>());
} finally {
  calloc.free(cstr);  // 确保释放内存
}
```

## 7. 安全机制

### 沙箱模式

```
启用沙箱前的 Lua 环境:
┌─────────────────────────────────────────────────────┐
│ os         → 操作系统 API (危险)                    │
│ io         → 文件 I/O (危险)                         │
│ package    → 模块系统 (危险)                         │
│ require    → 动态加载 (危险)                         │
│ load       → 动态编译 (危险)                         │
│ dofile     → 文件执行 (危险)                         │
│ string     → 字符串库 (安全)                         │
│ table      → 表库 (安全)                             │
│ math       → 数学库 (安全)                           │
└─────────────────────────────────────────────────────┘

执行沙箱代码:
os = nil
io = nil
package = nil
require = nil
load = nil
dofile = nil

启用沙箱后的 Lua 环境:
┌─────────────────────────────────────────────────────┐
│ os         → nil (已禁用)                           │
│ io         → nil (已禁用)                           │
│ package    → nil (已禁用)                           │
│ require    → nil (已禁用)                           │
│ load       → nil (已禁用)                           │
│ dofile     → nil (已禁用)                           │
│ string     → 字符串库 (可用)                         │

```

## 8. 异步处理

### 异步回调时序图

```
时间线:
t0: executeString("createNode(...)")
    │
    ├─ Dart: executeString() 开始
    │  Lua: 调用 createNode("Title", "Content", "callback")
    │
    ▼
t1: createNode API 被调用
    │
    ├─ Dart: createNode() 函数执行
    │  Dart: nodeRepository.save(node)  // 异步操作开始
    │  Dart: return 0  // 立即返回
    │
    ▼
t2: executeString() 返回
    │
    ├─ Dart: 返回 LuaExecutionResult
    │  输出: ["创建请求已提交..."]
    │  注意: 此时回调还未执行
    │
    ▼
t3: ... (异步操作进行中)
    │
    └─ Repository: 保存节点到文件系统
    │
    ▼
t4: 异步操作完成
    │
    ├─ Dart: save().then() 回调执行
    │  Dart: invokeCallback("callback", [true, {...}])
    │  Dart: 构建并执行 Lua 脚本
    │  Lua: callback(true, {...})
    │  Lua: print("创建成功!")
    │
    ▼
t5: 回调输出
    │
    └─ debugPrint("[LUA] 创建成功!")
```

**关键问题**:
- t2 时 `executeString()` 已经返回
- t4 时的回调输出不会出现在 `executeString()` 的结果中
- 这是设计上的权衡：Lua 同步模型 vs Dart 异步模型

## 9. 性能优化

### 双引擎架构选择

```
┌─────────────────────────────────────────────────────┐
│ SimpleScriptEngine                                   │
│ ┌───────────────────────────────────────────────┐ │
│ │ 优点:                                          │ │
│ │  ✓ 快速启动 (~1ms)                            │ │
│ │  ✓ 无外部依赖                                 │ │
│ │  ✓ 适合简单脚本                               │ │
│ │                                               │ │
│ │ 缺点:                                          │ │
│ │  ✗ 功能有限 (无表、闭包等)                    │ │
│ │  ✗ 性能较差                                   │ │
│ └───────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────┐
│ RealLuaEngine                                        │
│ ┌───────────────────────────────────────────────┐ │
│ │ 优点:                                          │ │
│ │  ✓ 完整 Lua 5.2 功能                          │ │
│ │  ✓ 高性能 (~10-50ms)                          │ │
│ │  ✓ 支持标准库                                 │ │
│ │                                               │ │
│ │ 缺点:                                          │ │
│ │  ✗ 启动稍慢 (~10ms)                           │ │
│ │  ✗ 需要 lua.dll                               │ │
│ └───────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────┘

使用建议:
- 开发/测试: SimpleScriptEngine (快速迭代)
- 生产环境: RealLuaEngine (完整功能)
```

## 10. 实际应用示例

### 批量节点处理

```lua
-- Lua 脚本
getAllNodes("processNodes")

processNodes = function(success, result)
  if success then
    local count = 0
    for i, node in pairs(result.nodes) do
      -- 重命名节点
      local newTitle = "Updated: " .. node.title
      updateNode(node.id, newTitle, nil, nil)
      count = count + 1
    end
    showMessage("已处理 " .. count .. " 个节点")
  end
end
```

执行流程:

```
1. getAllNodes() 调用
   │
   ├─ Dart: queryAll() 从数据库读取
   │  └─ 返回 100 个节点
   │
   ▼
2. processNodes 回调
   │
   ├─ Lua: 遍历 100 个节点
   │  └─ for i, node in pairs(result.nodes)
   │
   ▼
3. 批量更新
   │
   ├─ Dart: 100 次 updateNode() 调用
   │  └─ 每次都是异步 save()
   │
   ▼
4. 完成
   │
   └─ showMessage("已处理 100 个节点")
```

## 总结

Lua 插件的实现是一个**多层、复杂但优雅的系统**：

1. **FFI 层**: 连接 Dart 和 C/Lua
2. **引擎层**: 管理 Lua 运行时
3. **API 层**: 提供应用功能
4. **安全层**: 沙箱和权限控制
5. **类型层**: 自动类型转换
6. **内存层**: 指针和资源管理

这个实现充分利用了 Flutter/Dart 的系统集成能力，为应用提供了强大的脚本扩展功能！
