## 技术问题分析

### 问题 1: 异步回调输出捕获

**现象**:
```dart
// Lua 脚本
callback = function(success, result)
  print("回调被调用")  // 这行输出未出现在 executeString 结果中
end
createNode("Title", "Content", "callback")
```

**原因**:
1. `executeString()` 是同步执行，Lua 脚本执行完后立即返回
2. `createNode()` 内部的 `nodeRepository.save()` 是异步操作
3. 回调函数在 Future 完成后才执行，此时 `executeString()` 已经返回

**影响**:
- 不影响实际功能（Repository 方法被正确调用）
- 仅影响测试中的断言（无法验证回调输出）
- 用户在实际使用中可以通过其他方式（如 UI 更新）观察结果

**解决方案**:
1. **短期**: 修改测试，不依赖异步回调的同步输出
2. **中期**: 添加异步等待机制，支持 `executeStringAsync()`
3. **长期**: 实现 Future 与 Lua 协程的集成

### 问题 2: 沙箱权限配置

**现象**:
```
LuaSecurityException: 权限不足: nodeWrite
```

**原因**:
测试中启用了沙箱但未配置适当的权限

**解决方案**:
在测试中使用宽松的沙箱配置或不启用沙箱
